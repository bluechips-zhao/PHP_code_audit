  Step 2：串行攻击（逐个专家独占容器）
  ┌──────────────────────────────────────────────────────────────────────┐
  │  按 P0→P1→P2 优先级排序，逐个专家执行：                               │
  │  → 读取 _plan.json → 逐轮发送 Payload（默认最多 8 轮，可配置）        │
  │  → 每轮前 Docker 快照，失败后回滚                                      │
  │  → 成功采集物理证据（HTTP 响应/命令输出）                               │
  │  → 发现共享到 audit_session.db                                        │
  │  → 连续 N 轮无新发现 → 早停退出（early_stop_rounds，默认 2）          │
  └──────────────────────────────┬───────────────────────────────────────┘
                                 ↓
  Pivot When Stuck（连续失败自动转向）
  ┌──────────────────────────────────────────────────────────────────────┐
  │  sqli 全部失败  → 切换二阶 SQLi（追踪存储→读取→拼接路径）              │
  │  xss 被完全过滤 → 尝试 SSTI（{{7*7}} 探测模板引擎）                   │
  │  lfi 路径被过滤 → php://filter chain RCE                             │
  │  rce disable_fn → 反序列化 POP chain                                  │
  │  ssrf 内网不可达 → DNS Rebinding                                      │
  └──────────────────────────────┬───────────────────────────────────────┘
